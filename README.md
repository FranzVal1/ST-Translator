# SillyTavern Safe Translation

Расширение для SillyTavern 1.18.0+. Интеллектуально переводит текстовые сегменты сообщения через штатные маршруты Google, Yandex или Bing. 

**Главное отличие от аналогов:** Расширение парсит разметку и отправляет переводчику *только* чистый текст. HTML-теги, макросы `{{...}}`, блоки кода, URL, встроенные скрипты и служебные XML-блоки защищены от перевода. Благодаря пакетной обработке (батчингу) сохраняется контекст между абзацами, а скорость работы значительно выше, чем при постатейном переводе.

## 🌟 Основные возможности

<details>
<summary><b>Нажмите, чтобы развернуть список фич</b></summary>

*   **Поддерживаемые провайдеры:** Google, Yandex, Bing (через стандартные API endpoints SillyTavern).
*   **Безопасный парсинг:** Сохраняет структуру сообщения. Выделение кода, аватары, кнопки и верстка не ломаются.
*   **Пакетный перевод (Батчинг):** Текстовые блоки объединяются в один запрос (с учетом лимитов провайдера) через уникальные разделители. Это позволяет переводчику понимать контекст всего сообщения, а не отдельных предложений, и работает в разы быстрее.
*   **Защита служебных блоков:** Нестандартные теги вида `<instruction>...</instruction>` или `<[System]>` не переводятся целиком, если включена соответствующая опция.
*   **Перевод атрибутов:** Умеет переводить значения `title`, `alt`, `placeholder`, `aria-label` в стандартных HTML-тегах.
*   **Режимы работы:** Автоперевод ответов персонажа, ручной перевод кнопкой в меню сообщения, или перевод всего чата разом.
*   **Безопасное редактирование:** При клике "Редактировать" в поле всегда подставляется исходный `message.mes`, а не переведенный текст.
*   **Кэширование:** Результаты кэшируются в памяти для мгновенного отображения при перелистывании.
</details>

## 🚀 Установка

1. Скачайте архив или распакуйте папку плагина в `data/<пользователь>/extensions/SillyTavern-Safe-Translation`.
2. Перезапустите SillyTavern.
3. Откройте **Extensions → Safe Translation**.
4. ⚠️ **Важно:** Отключите автоматический режим встроенного плагина Chat Translation, чтобы два переводчика не конфликтовали.

## ⚙️ Технические детали и Архитектура

<details>
<summary><b>Как работает парсер и батчинг?</b></summary>

Парсер (находится в `parser.js`) токенизирует сообщение на "защищенные" сегменты (Protected) и "текстовые" (Text).
При включении батчинга все текстовые сегменты объединяются в единую строку вида:
`Текст 1\n[[ST_SEG]]\nТекст 2\n[[ST_SEG]]\nТекст 3`

Эта строка отправляется одним запросом. Переводчик видит весь текст и правильно согласует местоимения и термины. После перевода плагин разбивает ответ по разделителю `\[\[ST_SEG\]\]` и собирает исходную HTML-структуру.

**Размер чанка:** Алгоритм `splitForProvider` имеет жесткий fallback на случай, если текст не содержит пробелов (например, огромный URL, пропущенный парсером). Это исключает зависание скрипта в бесконечном цикле.
**Лимиты API:** $L_{max}$ равен 5000 символов для Google/Yandex и 1000 для Bing.
</details>

## 📋 Changelog

<details>
<summary><b>Version 1.2.0 (Текущая)</b></summary>

*   **Реализован Batched Placeholder Translation:** Текстовые сегменты переводятся одним/несколькими большими запросами с использованием разделителей. Скорость работы повышена в 5-10 раз. Контекст перевода между абзацами сохраняется.
*   **Исправлен Race Condition в редакторе:** Использование `MutationObserver` вместо каскада `setTimeout` для надежной подстановки оригинального текста в форму редактирования.
*   **Исправлен payload для Yandex API:** Приведен к стандартному виду `{ text, lang }` для совместимости с роутами SillyTavern.
*   **Исправлена ошибка разбиения:** Защита от зацикливания при разбиении строк без пробелов.
*   Улучшена обработка ошибок HTTP-запросов (парсинг JSON-ответа от ST).
</details>
